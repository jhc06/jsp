<JSP 퀴즈>

문제1> DNS는 무엇이며 어떤 기능을 합니까?
    - DNS는 Domain Name Server의 약자로 클라이언트가 브라우저 상에 입력한 도메인 주소에 대응하는 IP주소를 돌려주는 서버를 뜻합니다.
    예로는 주로 SKT,KT와 같은 통신사 DNS서버와 구글, 클라우드플레어 등 기업에서 DNS서버가 있습니다.
문제2> 클라이언트와 서버 개념을 설명해 보세요.
    - 식당에서 손님이 주문을 하면 식당에서 음식을 제공하듯이 웹상에서 클라이언트가 요청하는 데이터를 서버가 제공합니다.
    클라이언트와 서버는 TCP/IP나 UDP와 같은 통신 프로토콜을 통해 데이터를 주고 받는 통신을 하는 컴퓨터를 뜻합니다.
    좀 더 구체적으로는 클라이언트는 사용자가 쓰는 브라우저, 서버는 웹 서버를 뜻합니다.
문제3> 웹서버가 사용하는 기본 포트 번호는 80입니다. 포트번호를 주는 의미는 무엇일까요?
    - 통상 웹서버는 여러 프로그램을 운용하는데 클라이언트에서 요청하는 프로그램에 접속하기 위한 문과 같은 것입니다.
    서버를 구축 시 포트바인딩을 통해서 외부 접속을 할 수 있게 구축하며, 이미 타 프로그램에 사용되는 포트는 중복사용이 불가능합니다.
문제4> 렌더링(rendering)무엇을 뜻하나요?
    - 클라이언트의 요청에 서버가 응답한 결과를 웹브라우저 상에서 표시해주는 기능.
문제5> 웹서버에 따라 웹 프로그래밍이 달라지는데, 웹프로그래밍 기술은 무엇이 있나요? (3가지 정도)
    - 가장 기본적인 프로그램으로는 정적인 페이지를 만드는 HTML,CSS가 있으며, 앞의 것과 다르게 동적인 페이지를 만드는 PHP, 가장 많이 쓰이는 JavaScript, ruby와같은 프로그램이 있습니다.
문제6> HTTP 404, 500 에러의 의미는 각각 무엇인가요? **************
    - 404: 사용자가 요청한 도메인네임에 해당하는 페이지가 없을 경우를 뜻하는 에러(notfound)
    500: 웹서버 내부에서 발생한 에러로 요청한 페이지를 출력하지 못함을 뜻하는 에러(Internal server error)
문제7> JSP의 기본 객체들이 주로 하는 일들은 무엇인가요? (request, response, session 등)
    - request란 클라이언트가 get이나 post 방식으로 요청하는 데이터를 저장하는 객체이고, 반대로 response는 클라이언트의 요청에 따른 웹서버에서 처리한 결과를 보내는 객체입니다.
     session은 클라이언트와 웹서버간의 연결을 유지시켜주는 일을 합니다. 예를 들면 사용자가 로그인을 하였을 때 처음 요청 후 다음 요청 시에도 같은 사용자가 요청했다는 사실을 확인하여
     기존의 정보를 활용할 수 있게 합니다.
문제8> MIME(Multipurpose Internet Mail Extensions)을 JSP의 contentType에 기술하는 이유는 무엇인가요? (예: text/html, application/json)
    - JSP에서 contenttype을 기술하여 웹서버에서 제공하는 데이터를 클라이언트인 브라우저가 해석할 수 있게 합니다.
문제9> JSP에서 <%@ page trimDirectiveWhitespaces="true" %> 를 기록하는 이유는 무엇인가요?
    - 페이지 trimdirective는 JSP문서를 작성할 시 브라우저상에 출력 시 소스 상 JSP문이 들어가는 부분이 빈 공백으로 남는것을 제거하여 코드를 깔금하게 만들어주는 역할을 합니다.
문제10> JSP에서 <% %>, <%= %>, <%! %>의 기능을 각각 설명하세요.
    - 꺽세 플러스 퍼센트는 JSP문상 스크립트릿이라고 하며, JSP 문서에서 자바 코드를 입력할 수 있는 코드블록을만듭니다. 꺽세 플러스 퍼센트 이퀄은 표현식이라고 하며, 어떤 값을 출력결과에 포함시킬 떄 사용합니다.
    꺽세 퍼센트 느낌표는 선언부라고 하여 스크립트릿이나 표현식에 사용할 메소드를 작성시 사용합니다.

문제13>HTTP 프로토콜의 GET방식과 POST방식을 비교해서 설명해 보세요.
    - get방식과 post방식을 둘 다 http메소드로 데이터를 전송할 때 쓰이는 방식입니다.
    Get은 전송할 데이터를 url에 도메인네임?뒤에 입력하여 서버에 전송합니다. 이 방식은 url에 데이터가 입력됨으로 데이터 크기에 제한이 있고 보안상 취약하다는 단점이 있지만, post 방식에 비해
    속도가 빠르다는 장점이 있습니다.
     반면, Post방식은 요청 데이터를 body문에 입력하여 전송합니다. 이 방식은 get방식에 비해 데이터가 body에 입력됨으로서 보안상 안전하다는 장점이 있지만, get 방식에 비해 속도가 느리다는 단점이 있습니다.
     웹페이지 작성 시 어느 하나가 우위에 있는 것이 아니라 보안이나 속도를 고려하여 작성해야합니다.
문제14>GET방식으로 파라미터를 전송하는 3가지 방법을 설명하세요.
    - 첫번째로 URL에 도메인 네임? 뒤에 파라미터를 입력하는 방법, 둘째로 form을 통해서 입력하는 방법, <a> 태그의 링크 태그를 통해 입력하는 방법이 있습니다.
문제15>response 기본 객체가 하는 일 2가지를 설명하세요.
    - request, response, pageContext, Session, application, out, config, page, exception, 총 9가지.
    out: jsp 페이지 생성 결과를 출력 시 사용하는 출력 스트림.
    config: jsp 페이지에 대한 설정 정보를 저장
    page: jsp 페이지를 구현한 자바 클래스 인스턴스
    exception: 에러 페이지에서 사용함
문제16>웹브라우저의 종류를 아는데로 써 보세요. (3가지 이상)

문제17>웹브라우저가 하는 캐시(Cashe) 기능은 무엇입니까?
    - 캐시는 서버와 클라이언트 간의 통신 데이터 중 클라이언트 측에 저장되는 데이터입니다.
     캐시가 없다면 둘 간의 통신이 매번 새로 데이터를 받으면 서버에 부담을 주며 데이터 전송량이 늘어나 통신망에 부담을 줍니다.
     이를 막기위해서 클라이언트에 필요한 데이터를 저장해놓아서 새로운 요청 시 이를 활용하게 합니다.
문제18>WAS는 무엇의 약자입니까?
    - WAS Web Appication Server의 약자입니다.
문제19>JSP 페이지에 대한 요청 처리 과정을 설명하세요. *******************

문제20>출력버퍼의 플러시(flush)는 어떤 의미인가요?
    - 출력버퍼에 저장된 데이터를 전송을 위한 장소로 옮기는 것을 뜻합니다. 플러시 되면 데이터를 수정할 수 없습니다.
문제21>JSP 페이지를 처리해서 전송할 때에 버퍼를 왜 사용할까요? *******************
    - 데이터 전송 성능 향상, JSP 실행 도중에 버퍼를 비우고 새로운 내용 전송 가능, 버퍼가 다 차기 전까지 헤더 변경 가능.
문제22>웹어플리케이션 배포는 무슨 뜻인가요?
    - 웹 어플리케이션을 WAS에서 사용할 수 있게 파일을 올리는 것으로 직접 파일 업로드와 WAR 파일로 만들어 업로드 하는 두가지 방법이 있습니다.
